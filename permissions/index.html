3. 외부 시스템 쓰기 권한 (시크릿 위치)

모든 시크릿은 ~/claude_bots/{bot}/.env에 평문 저장. .gitignore로 git 추적 차단. 회전 시 .env 편집 후 봇 재시작 트리거.

4. 민감 동작 안전장치
텔레그램 파일 송신 (send_dispatcher.py)
Cowork 디스패치가 send_queue/pending에 파일+메타 드롭 → 워커가 Telegram sendDocument.
chat_id 강제 화이트리스트: ALLOWED_GROUP_CHAT_IDS(이사회) + ALLOWED_USER_IDS(석만) 외에는 발송 거부, failed/로 이동. 실수로 외부 채널에 못 감.
개인 도구 게이트 (secretary_bot)
publish_analysis, github_publish_html은 _require_admin() 함수가 호출자 user_id를 ADMIN_USER_ID와 대조. 불일치 시 즉시 거부 메시지("⛔ 관리자 본인만") 반환. contextvars.ContextVar로 호출자 user_id를 핸들러 진입 시 세팅.
현재 미구현 — 파괴적 동작
파일 삭제 · GitHub 파일 삭제 · 캘린더 이벤트 삭제 등 되돌릴 수 없는 동작은 아직 도구로 노출 안 함. 추후 추가 시 명시적 확인 패턴(/confirm_delete <id>) 별도 구현 예정.
5. 권한 변경 절차
사용자 추가 (1:1 화이트리스트)
  1. 대상자에게 봇한테 /start 1번 보내달라고 요청
  2. 봇 로그에서 user_id 확인 (또는 @userinfobot 활용)
  3. ~/claude_bots/{bot}/.env의 ALLOWED_USER_IDS에 콤마 구분으로 추가
  4. echo "{bot명}" > ~/claude_bots/.restart_trigger
단톡방 추가 (claude_bot 그룹 화이트리스트)
  1. 봇을 단톡방에 초대 + 권한 부여
  2. 단톡방에서 누군가 메시지 1번 보내면 chat_id가 claude_bot/chat_history.db에 기록됨
  3. SQLite 조회: sqlite3 ~/claude_bots/claude_bot/chat_history.db "SELECT DISTINCT chat_id, chat_title FROM messages"
  4. ~/claude_bots/claude_bot/.env의 ALLOWED_GROUP_CHAT_IDS에 추가
  5. echo "claudebot" > ~/claude_bots/.restart_trigger
시크릿 회전 (토큰 노출 의심 시)
  1. 외부에서 새 토큰 발급 (BotFather / GitHub Settings / Tavily 콘솔)
  2. ~/claude_bots/{bot}/.env 직접 편집
  3. 해당 봇 재시작 트리거
  4. 옛 토큰을 외부에서 즉시 revoke
