1. 에이전트란?
정의: LLM(두뇌) + 도구(tools) + 루프(loop) = 에이전트.
단순 챗봇이 "질문 → 답변 → 끝"이라면, 에이전트는 "결정 → 도구 호출 → 결과 관찰 → 다시 결정"을 자율적으로 반복해서 목표를 달성하는 시스템.
에이전트 루프 (agent loop)
사용자 메시지
↓
LLM 두뇌현재 상태 분석 + 다음 행동 결정
↓
도구 호출검색·DB조회·파일쓰기·API호출 등
↓
결과 관찰도구 결과를 두뇌에 다시 전달
↻ 목표 달성까지 반복
최종 답변목표 충족 시 사용자에게 전달

옥천당의 비서봇·원료봇·claude_bot은 모두 이 패턴을 따른다. Cowork(디스패치)도 같은 패턴의 에이전트이다.

3. 비서봇 내부 구조

비서봇은 자기 안에 8개 도구를 가진 라우터 에이전트. haiku가 사용자 의도를 파악하고 적절한 도구를 호출. 도구 결과를 받아 다음 행동을 결정하는 루프를 최대 8회 반복.

4. B 패턴 — Cowork 직접 발행 (API 크레딧 0원)

Cowork(디스패치)는 Claude Max 구독 토큰을 사용하므로 동일한 작업을 비서봇 대신 직접 처리하면 Anthropic API 호출 비용이 발생하지 않는다. 단, GitHub API 호출은 Cowork 샌드박스에서 막혀 있어 호스트의 publisher.py를 거쳐야 한다. 이를 위해 파일 큐(publish_queue) + launchd watcher 구조를 둠.

왜 이 구조가 필요한가
Cowork 샌드박스는 외부 인터넷이 allowlist 기반(Anthropic·npm·pip·github.com 등)이라 api.github.com·api.telegram.org가 막혀 있음. publisher.py는 호스트 Mac에서 launchd로 실행돼 그런 제약 없음. 큐 파일을 매개로 두 영역(샌드박스 vs 호스트)을 분리해 보안 모델을 유지하면서도 작업이 흐르게 함.
5. 권한 모델 (하이브리드 B안)
공개 도구 정보 검색·분석 도구 (raw_materials, business_chat, deep_analyze, web_search, literature_search, kor_literature_search). 단톡방 멤버 누구나 사용 가능. 사용자 본인 외부 계정·시스템에 쓰기 동작 없음.
개인 도구 외부 시스템 쓰기 (github_publish_html, publish_analysis). _require_admin() 게이트가 호출자 user_id를 ADMIN_USER_ID(22191413)와 대조. 불일치 시 즉시 "⛔ 관리자 본인만" 거부 메시지.
관리자 검증 contextvars.ContextVar로 호출자 user_id를 핸들러 진입 시 세팅. asyncio.to_thread로 동기 코드 위임 시에도 contextvar 자동 전파.
파괴적 동작에 대한 추가 보호
현재는 등록·생성만 가능. 향후 삭제(GitHub 파일 삭제, 캘린더 이벤트 삭제 등)를 추가할 때는 명시적 확인 패턴(/confirm_delete <id>)을 따로 박을 예정.

코드 수정 후 봇 재시작은 Cowork에서 트리거 파일에 한 줄 echo로 끝남. 사용자가 Terminal 켤 필요 없음.

echo "secretary" > ~/claude_bots/.restart_trigger
echo "claudebot" > ~/claude_bots/.restart_trigger
echo "rawbot"    > ~/claude_bots/.restart_trigger
echo "all"       > ~/claude_bots/.restart_trigger
macOS TCC 회피

launchd로 실행되는 프로세스는 ~/Documents·~/Desktop·~/Downloads(TCC 보호 영역)의 파일을 읽지 못한다. 봇·인프라·큐 폴더는 모두 홈 디렉터리 루트 아래(~/claude_bots/)에 배치하여 회피.

7. 다중 에이전트 중첩 — "에이전트의 에이전트"

발행 한 건이 일어날 때 여러 에이전트·LLM이 협업한다. 예: 사용자가 비서봇에 "갈근 임상근거 분석 페이지 발행해줘" 요청 시.

중첩 구조 예시
사용자텔레그램 메시지
↓
비서봇 라우터 (haiku)의도 파악 → 도구 선택
↓ 도구 호출
kor_literature_searchTavily(한국 학술) — KCI/RISS 결과 회수
＋
literature_searchPubMed — 영문 논문 회수
↓ 결과를 context로 전달
publish_analysis 도구내부에서 두 단계 실행
↓
(중첩) deep_analyze (sonnet)풀 HTML 페이지 생성
↓
GitHub API PUT파일 등록
↓
okri.ocd.kr URL 반환비서봇 라우터로 복귀
↓
비서봇 라우터 (haiku)"✅ 발행됨: URL" 답변 생성
↓
사용자에게 답변

즉 한 요청에 LLM 호출이 4~5회 발생: haiku 1~2회(라우팅) + sonnet 1회(분석·HTML 생성). 검색 도구는 LLM이 아닌 외부 API(Tavily/PubMed) 호출.

8. 청구 모델 — 두 경로의 비용 차이
경로비용 모델건당 평균유리한 케이스
비서봇 Anthropic API 토큰 (per-token 과금) $0.05~$0.20 모바일·외부·단톡방·즉답 필요 케이스
Cowork 디스패치 + publish_queue Claude Max 구독 (월 정액) $0 (구독 한도 내) 책상 앞에서 직접 작업, 깊은 사고 필요한 경우

한 요청을 어느 경로로 흘릴지는 사용자가 결정. 똑같은 도구·결과물(okri.ocd.kr 페이지)이지만 누가 LLM 호출을 하느냐에 따라 청구처가 다름.

9. 향후 확장 후보
자율성 확장
  • 스케줄 기반 자동 발행 (예: 매주 시장 동향 페이지 갱신)
  • 외부 이벤트 트리거 (메일·드라이브 업로드 → 자동 분석)
도구 확장
  • Notion 저장 (개인 도구)
  • Google Calendar 등록 (개인 도구, OAuth)
  • Google Drive 읽기 (개인 도구, OAuth)
  • 옥천당 ERP·내부 DB 연동
전문 에이전트 분리
  • 법규 검토 에이전트 (식품위생법·약사법·건기식법 전담)
  • R&D 기획 에이전트 (임상시험 설계·약리 가설 전담)
  • 마케팅 에이전트 (한의사 대상 컨텐츠 작성 전담)
  • 비서봇이 라우터로 각 전문가에게 위임
