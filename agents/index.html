1. 에이전트란?
LLM(두뇌) + 도구(tools) + 루프(loop) = 에이전트.
단순 챗봇이 "질문 → 답변 → 끝"이라면, 에이전트는 "결정 → 도구 호출 → 결과 관찰 → 다시 결정"을 자율 반복해 목표를 달성합니다.

6. Cowork 직접 발행 — B 패턴

Cowork 디스패치는 Max 구독 토큰을 사용해 봇 비서 대신 직접 처리할 수 있음. 단 GitHub API · Telegram API가 샌드박스에서 차단되어 호스트의 워커를 거쳐야 함. 이를 위해 파일 큐 + launchd watcher 구조.

B 패턴 파이프라인
① Cowork (제 사이드)검색 · 분석 · HTML 작성 (Max 토큰)
↓
② publish_queue/staging<id>.html + <id>.json
↓ atomic mv
③ publish_queue/pendinglaunchd watch
↓ 자동 트리거
④ publisher.py (호스트)stdlib만 사용 · GitHub PUT
↓ 동시 분기
⑤a GitHub APIcontents PUT (sha 갱신)
⑤b Telegram API관리자에게 알림
↓
⑥ processed/<id>.urlCowork이 폴링해 URL 회수
↓
okri.ocd.kr / ...
동일 패턴이 텔레그램 발송에도 적용
send_queue (~/claude_bots/claude_bot/send_queue/) + send_dispatcher.py + com.okcheon.board_sender. Cowork이 파일 + meta JSON 드롭하면 launchd가 Telegram sendDocument 호출. chat_id 화이트리스트로 실수 발송 방지.
